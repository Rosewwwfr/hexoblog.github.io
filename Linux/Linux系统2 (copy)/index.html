虚拟机配置固定IP

为何虚拟机的IP会变？

因为其IP地址是通过DHCP服务获取的。

DHCP:动态获取IP地址，每次重启都会获取一次。

步骤：

  • 配置网关和网段（IP地址的范围）
  • 手动修改配置文件。

裂开，配置了后没变。

检查指定的网络服务是否为可联通状态。

选项：

  • -c 检查的次数 ，不设置为无限次。

wget

非交互式文件下载器，在命令行下载网络文件。

-b 后台下载，间日志写进但钱工作目录的wget-log文件

curl

选项：

-0 用于下载文件，保存文件。

端口

设备于外界通讯交流的出入口。

可分为：

  • 物理端口
    • 平时见到的那些口。
  • 虚拟端口
    • 用于操作系统和外部进行交互。

IP只能锁定计算机，用端口锁定计算机上的具体的程序。

也可以比作IP地址是小区地址，住户的门牌号就像端口。

Linux系统端口分类

支持65535个

  • 公认端口：1~1023 系统预留。 如22为SSH端口。
  • 注册端口：1024-49151任意使用.
  • 动态端口：49152~65535 ，进行网络连接时用到。

进程管理

程序运行时，被操作系统管理，注册为系统的一个进程。

会分配一个进程ID、

查看进程

选项：

  • e 显示全部进程
  • f 格式化形式。

从左到右分别是：

  • UID：进程所属的用户IDPID：进程的进程号ID
  • PPID：进程的父ID（启动此进程的其它进程）
  • C：此进程的CPU占用率（百分比）
  • STIME：进程的启动时间TTY：启动此进程的终端序号，如显示?，表示非终端启动
  • TIME：进程占用CPU的时间
  • CMD：进程对应的名称或启动路径或启动命令

环境变量

作用：操作系统运行时，记录一些关键性的信息，以辅助系统运行。

环境变量是一种KeyValue型结构，即名称和值

环境变量PATH

用冒号分隔开。

记录了系统执行命令时的收索路径

/usr/local/bin

/usr/bin

/usr/local/sbin

/usr/sbin

当执行任何命令，都会按照顺序，从上述路径中搜索要执行的程序的本体

比如执行cd命令，就从第二个目录/usr/bin中搜索到了cd命令，并执行

$符号

压缩、解压

Linux常用的压缩格式

  • .tar
  • .gz

tar

压缩或者解压tar或者gzip文件

语法：

  • -c，创建压缩文件，用于压缩模式
  • -v，显示压缩、解压过程，用于查看进度
  • -x，解压模式
  • -f，要创建的文件，或要解压的文件，-f选项必须在所有选项中位置处于最后一个
  • -z，gzip模式，不使用-z就是普通的tarball格式
  • -C，选择解压的目的地，用于解压模式

压缩：

注意：

  • -z选项如果使用的话，一般处于选项位第一个-
  • f选项，必须在选项位最后一个

解压：

-C选项单独使用，和解压所需的其它参数分开
