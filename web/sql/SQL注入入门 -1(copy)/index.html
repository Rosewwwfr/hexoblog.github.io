SQL注入基础

一个重要的库

information_schema 数据库

mysql 5.0版本之后产生了一个虚拟数据库information_schema，在这个数据库中有三个很重要的表

  • information_schema: 该数据表存储了mysql数据库中的所有数据库的库名
  • information_schema.tables： 该数据表存储了mysql数据库中的所有数据表的表名
  • information_schema.columns: 该数据表存储了mysql数据库中的所有列的列名

存储着数据库中的库名、表名以及字段名。

其中的表：

  • tables 存放着 表名，字段名是table_name
  • columns 存放着 字段名 ，字段名为 columns_name

table_schema 是tables表中的一个字段信息，对应着一个个数据库。可以用where 来看。

基础的sql操作

增删改查

  • show databases; 查看有哪些数据库
  • use mysql; 进入数据库
  • show tables; 查看数据库中有哪些表
  • select * from user; 查看users表中的所有内容
  • select host,id,name from mysql.user； 查看表中的指定字段
  • desc 表名 查看表中的字段

函数

group_concat() 联合记录，用，隔开。

concat 也是联合查询。

limit

限制输出的行数

基础思路

通过上方的库，查找到有什么库，有什么表，有什么字段。

再查询具体表，字段对应的信息。

  • 查看当前数据库中有哪些表。

    • ```
      id = -1 union select 1, group_concat(table_name) from information_schema.tables where table_schema = database()

  • 最后查询有表中的信息即可。

数字型注入和union注入

数据库查询语句，注入点在WHERE

就是不需要进行额外的操作就可以进行注入了。

这里一般情况下，是不可能回显很多行的，就需要通同limit去限制输出第几行。

或者通过 id =-1 或者很多的值，使之回显不了，从而回显我门想要的数据。

判断方法：

通过输入 2 和 3-1 看回显是否相同。如果不同就不是数字型。

字符型

在数据外包裹 单引号或者双引号。

所以就需要闭合前面的引号，然后注释（或者闭合）掉后面的引号。

注释用需要用编码格式。

常构造

布尔盲注

就是如果查询语句正确，就会回显东西。如果flase就会显示空白。

需要用到的函数

  • substring() 、mid() 、substr()

三个函数的功能基本是一样的。

都是两个参数。

substr(n1,n2)

  • n1为从第几位开始截取。
  • n2为截取几位。

结合二分法。进行信息的判断。

时间盲注

就需要用到if函数（或者and或者or的短路特性）和sleep()函数

报错注入

有些网站，在 sql语句错误的时候，会输出错误信息。

可利用利用函数

updatexml 第二个参数应该是XPATH路径，否则就会在引发报错的同时，将传入的参数进行输出。

堆叠注入

目标开启多语句执行的时候，可以修改数据库的任意结构和数据。

LIMIT（晕晕）

只能是数字型的。

判断方法：通过修改数字，会显示更多或者更少的记录数。

利用方法：

  • 如果没有order by 关键字的话，可以使用UNION注入

  • MySQL5.6前可以使用PROCEDURE 尝试注入

如果有写入权限，还可以使用 into outfile 语句向WEB目录写入webshell。

INSERT注入

插入数据表记录的语句。

tbl_name

注释掉后面的值，写入自己想要写入的值。比如向管理员表插入信息。

value

先闭合前面的引号，再插入想要插入的数据。

一般情况下管理员和用户同同一个表。

加入第二个是管理员标识，便可以插入一个管理员用户。

如果有数据会回显，也可以将查询语句插入到那个位置。

UPDATE注入

用于数据库记录的更新。

就比如说id 可控，就可以修改数据。如果可以回显，就直接插入查询语句。

where注入点则和前面的一样。

注入防御

字符替换

  • %09
  • %a0 只有在特定的字符集中才能利用。

  • 括号
