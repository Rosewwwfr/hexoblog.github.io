文件上传总结

发表于2023-05-14|更新于2023-05-14|web文件上传
|阅读量:

回过头来总结一下文件上传，学到了不少之前没注意到的知识点，但是仍有一部分不是很懂。

也会有一些没有在做题时用过。

用的最多的还是 .user.ini

文件上传

介绍：

  • 文件上传是通过流的方式将文件写到服务器上。
  • 文件上传必须以POST提交表单
  • 表单中需要 <input type="file" name="upload">

在渗透测试或者 ctf 过程中，遇到文件上传常见的思路无非是尝试绕过一些限制直接上传 shell （脚本文件）

一句话木马

<?php eval($_POST['a']);?>

通过上传一句话木马，执行任意命令，或者是蚁剑连接。

文件上传类型

  1. js前端验证
  2. 文件头检查
  3. .htaccess
  4. MIME绕过
  5. 检测后缀名
  6. 检测 文件内容（php）
  7. .user.ini
  8. log
  9. PNG图片二次渲染绕过
  10. zip文件上传

注意：

exif_imagetype() 函数检测文件类型。检查文件头

js前端验证

  • 关掉js验证
  • burp抓包 上传

文件头验证

就是文件的头部会有该文件的一些信息，在上传文件的时候，会对文件头进行检测。

绕过方法：

  • 直接画图，然后末尾加入一句话木马
  • 然后抓包，修改名字即可（即要保证php能够被解析）

.htaccess

介绍：

这是一个配置文件，说明对文件进行的操作。

.htaccess文件是Apache服务器中的一个配置文件，它负责相关目录下的网页配置，通过.htaccess文件可以实现网页301重定向、自定义404页面、改变文件扩展名、允许/阻止特定的用户或者目录的访问、禁止目录列表、配置默认文档等功能。参考

注意：

需要注意，.htaccess文件的作用域为其所在目录与其所有的子目录，若是子目录也存在.htaccess文件，则会覆盖父目录的.htaccess效果。

注意：**.htaccess 中有 # 单行注释符，且支持 \ 拼接上下两行**

利用：
  • 所以我们可以修改.htaccess文件，让其对我们上传的文件，进行指定操作。

  • 也可以把直接包含到php文件中。

    • 但是如果同目录没有php文件，可以把直接当作是php文件使用。

绕过黑名单

处理限制 .htaccess外，还可能限制关键字。

绕过对关键字的过滤我们可以使用反斜杠 \ 加换行来实现。

MIME绕过

MIME (Multipurpose Internet Mail Extensions) 是描述消息内容类型的因特网标准。
MIME 消息能包含文本、图像、音频、视频以及其他应用程序专用的数据。

即在传输过程中标记文件类型的一种方法，也就是 HTTP 文件请求头中的 Content-Type 。

说白了就是修改 类型信息。

双写绕过

当上传php 没有限制时，但是上传后php 又丢失了，就可以用 pphphp后缀来绕过。

大小写绕过后端

后缀大小写是通过服务端未将后缀进行统一的格式转换，导致可以上传PHP的文件，同时由于Windows系统对后缀大小写并不敏感，所以当在写PHP的改成Php就会被当作PHP进行解析。

00截断

后端只判断了后缀名，上传到服务器后%00后的字符串被截断。

www.xxx.com/qq.php%00.jpg => www.xxx.com/qq.php

利用条件：

  • PHP版本小于5.3.4
  • magic_quotes_gpc=Off，否则%00这种空字符会被转义为\0

注意 ：

GET 可以直接利用，但是POST 需要对%00进行一次URL编码。

双文件上传

在源码中隐藏着一个表单，通过修改这个表单（这个表单没有检测），上传我们要的文件。

即第一个表单上传 正常的，第二个上传木马。

Nginx CVE-2013-4547 漏洞

不太懂。

IIS6.0 目录路径检测解析绕过

IIS6.0 站上的目录路径检测解析绕过上传漏洞，当我们使用的服务器都是 Windows2003，并且使用的服务为 IIS6.0 时，就可能存在。

设置文件路径名为“uploading/234.php;”，文件名为“234.png”，由于 IIS 服务器会从头部查找查找”.”号,获得“uploading/234.php;234.png”。

接着查找到”;”，有则内存截断，所以这个请求会当做“uploading/234.php”进行解析。

通过这个方法，我们就可以将图片马以“php”为后缀保存在服务器上。

.user.ini

介绍：

自 PHP 5.3.0 起，PHP 支持基于每个目录的 .htaccess 风格的 INI 文件。此类文件仅被 CGI／FastCGI SAPI 处理。此功能使得 PECL 的 htscanner 扩展作废。

如果使用 Apache，则用 .htaccess 文件有同样效果。

原理：

.user.ini中两个中的配置就是auto_prepend_file和auto_append_file。

这两个配置的意思就是：我们指定一个文件（如1.jpg），那么该文件就会被包含在要执行的php文件中（如index.php），相当于在index.php中插入一句：require(./1.jpg)。

这两个设置的区别只是在于auto_prepend_file是在文件前插入，auto_append_file在文件最后插入。

利用.user.ini的前提是服务器开启了CGI或者FastCGI，并且上传文件的存储路径下有index.php可执行文件。（同路径）（这也是前提！！）

利用：

当后端 禁用 php 以及没有 "AddType application/x-httpd-php .php .phtml"时，需要利用这个来使图片shell 上传并被利用。

::$$DATA绕过

在window的时候如果文件名+::$DATA会把::$DATA之后的数据当成文件流处理,不会检测后缀名，且保持::$DATA之前的文件名，他的目的就是不检查后缀名

PNG图片二次渲染绕过

PNG图片二次渲染配合文件包含漏洞原理：
将一句话木马插入到网站二次处理后的图片中，也就是把一句话插入图片在二次渲染后会保留的那部分数据里，确保不会在二次处理时删除掉。这样二次渲染后的图片中就存在了一句话，再配合文件包含漏洞获取webshell。
直接在图片中插入一句话会被删除掉，二次渲染会把上传到服务器的图片的代码改变，
利用脚本构造出一个png木马图片

然后就会生成一个二次渲染过，仍有效的图片马。

然后访问 图片，执行命令。

当然，还是得配合 配置文件进行实现的。

zip文件

就是可以上传一个zip文件，写入一句话木马。

然后上传用蚁剑连接。

包含log 日志

当 命令执行完全被ban 如 web160

就可以用结合文件包含来使用。

include"/var/log/nginx/access.log"

如果被禁用了log，可以用字符串拼接来使用。

include"/var/lo"."g/nginx/access.lo"."g"
